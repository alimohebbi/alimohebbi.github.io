  • 2019 - now

    PhD Studies Software Engineering

    USI Università della Svizzera italiana, Lugano, Switzerland

    PhD Theses: Automated testing of interactive applications - Supervisor: Mauro Pezzè, Cosupervisor: Valerio Terragni

    Key Areas: GUI Testing • NLP • Mobile Applications Testing • Cloud Testing

    The goal of my Ph.D. is to automatically generate GUI test cases that are semantically relevant to the target application. That means a test case exercises a functionality of an application in a way that developers intended to be used. In my Ph.D., I focus on test reuse techniques that can generate semantically meaningful test cases by migrating a test case from one application to another. To do so test reuse approaches find events in the target application which are semantically equivalent to events of the source test case. Some highlights of my research so far are:

    • We designed a new algorithm for semantic matching of GUI events that improved the state-of-the-art by 7.5 %
    • We built a new train set of applications description by crawling Google Play. We used the new train set to build word embedding models which As a result improved semantic matching by 2%
    • We investigated the effectiveness of domain-specific trains sets for word embedding by applying Università della Svizzera topic modeling
    • We created and executed more than 9000 GUI test cases automatically to study the impact of semantic matching. We used 28 google cloud virtual machines with nested virtualization (docker) in parallel to conduct our experiments

    During my Ph.D., I also worked on PREVENT research project. Prevent is a fault prediction and localization approach in cloud systems. Our new proposed approach uses Autoencoder - a type of Deep Learning algorithm - to find anomalies in system health metrics. We evaluated the approach on a cluster of Redis (key-value database) nodes. We used the ELK stack to collect system health metrics from 20 nodes of a Redis cluster. Results of experiments showed False alarms were reduced by 30% and true positives improved by 40% in comparison to the state of the art.

  • 2016 - 2018

    Master Studies Software Engineering

    Sharif University of Technology, Tehran, Iran

    Master Theses: Bug prediction based on mutation metrics

    Key Areas: • Mining Software Repositories • Machine Learning

    Predicting bugs in software artifacts based on mutation scores is a new research direction. In my thesis, we proposed new metrics based on historical mutation analysis to predict bugs. Proposed metrics are extracted based on analysis of different versions of a software system. To evaluate the metrics empirically we used defect4j , a well-known data set, to conduct our experiment. The data set contains 6 open source JAVA projects repositories and their bugs. We mined repositories of the projects and extracted the metrics. We build prediction models from the extracted metrics by leveraging four classification techniques including: Neural Network, SVM, Logistic Regression and Decision Tree. Models predict if a java file will contain a bug in the next release. Models created with the proposed metrics improved by 6.6% in comparison to previous models in terms of accuracy.

Projects

Projects

Domain Specific Word Embedding

This project provides a word embedding model for a given document based on the domain of the document. The project uses topic modeling approaches to create clusters of documents. For each clusters it creates word embedding models. When a new document is queried it identifies an appropriate cluster (domain) for the document and provides a word embedding model built based on that cluster. The project support 3 clustering approaches (LDA, SLA, HDP) and 4 word embedding techniques (FastText, WordMovers, GloVe, Word2Vec) . Pretrained models available in the project repository are trained on data set of ~ 1 million english description of android applications in Google Play store.

Repository

Semantic Matching Framework

Semantic Matching framework provides ability to evaluate different methods of semantic matching for GUI events. Semantic matching is a process of finding a match for a given text among multiple target candidate texts. The framework operates in the context of Test Reuse of GUI application, however, it can be extended to be used in any context that leverage semantic matching. Currently the framework offers 273 different configuration. For a given data set of GUI events framework evaluates performance of different semantic matching configurations.
